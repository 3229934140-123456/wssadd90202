## 1. 产品概述

连锁医美门店术后忌口内容配置后台，面向运营主管和项目医生，将各项目的饮食规则统一管理、按分类维护、支持个性化医嘱与门店差异化下发，确保忌口话术版本可控、顾客疑问可追溯、运营效果可度量。

- 核心目标：消除各门店术后忌口话术不一致、版本混乱、顾客高频追问无反馈闭环的问题
- 目标用户：运营主管（方案配置、门店下发、效果追踪）、项目医生（个性化医嘱、问答维护）

## 2. 核心功能

### 2.1 用户角色

| 角色 | 入口方式 | 核心权限 |
|------|----------|----------|
| 运营主管 | 管理后台账号登录 | 方案增删改查、门店下发、小程序文案配置、效果看板查看、问答库管理、审批优化建议 |
| 项目医生 | 管理后台账号登录 | 方案查看、添加个性化医嘱、问答库维护、提交优化建议 |

### 2.2 功能模块

1. **方案库**：按项目分类维护忌口方案，含禁忌食物、建议食物、提醒频率、恢复阶段、特殊人群说明
2. **项目分类**：按皮肤管理、微整注射、外科手术、激光光电等分类管理项目
3. **医嘱编辑**：医生给项目添加个性化医嘱，支持富文本和标签式编辑
4. **门店下发**：方案下发到指定门店，支持区域差异化保留
5. **问答库**：顾客常问问题整理成问答，关联到具体项目
6. **效果看板**：总部查看各门店启用率、阅读率、风险反馈量

## 3. 核心流程

**方案创建与下发流程**：运营主管按分类创建忌口方案 → 填写禁忌/建议食物、提醒频率、恢复阶段、特殊人群说明 → 医生添加个性化医嘱 → 运营选择门店下发 → 门店启用方案 → 顾客在小程序查看忌口内容

**版本管控流程**：方案修改 → 填写修改原因和生效时间 → 系统记录修改人和时间 → 新版本下发门店 → 旧版本标记为已过期

**反馈优化流程**：门店收集顾客高频追问 → 提交优化建议 → 运营审批 → 方案更新 → 重新下发

## 4. 用户界面设计

### 4.1 设计风格

- 主色调：深青色 (#0F766E) + 暖金色 (#D4A853) 点缀，传达专业医美与高端感
- 辅助色：浅灰白背景 (#F8FAFB)，卡片白 (#FFFFFF)，危险红 (#DC2626) 用于禁忌标注
- 按钮风格：圆角 8px，主按钮填充深青色，次按钮描边，危险操作红色
- 字体：标题使用 Noto Serif SC（宋体雅致感），正文使用 Noto Sans SC，数据看板使用 DIN Alternate
- 布局：左侧固定导航栏 + 顶部面包屑 + 右侧内容区，卡片式内容组织
- 图标风格：线性图标（Lucide 风格），24px，与文字同色

### 4.2 页面设计概览

| 页面名称 | 模块名称 | UI 元素 |
|----------|----------|----------|
| 方案库 | 方案列表 | 顶部搜索栏+分类标签切换、卡片网格布局、每张卡片含分类色带、状态标签、快捷操作按钮 |
| 方案库 | 方案编辑 | 左侧表单区域、右侧实时预览手机效果、禁忌食物红色标签、建议食物绿色标签 |
| 项目分类 | 分类管理 | 分类卡片横向排列、每类含图标和项目数量、拖拽排序 |
| 医嘱编辑 | 医嘱编辑器 | 富文本工具栏、标签式医嘱模板、右侧项目关联面板 |
| 门店下发 | 下发配置 | 左侧方案选择树、中间门店列表（可按区域筛选）、右侧下发预览、底部确认操作栏 |
| 门店下发 | 小程序文案配置 | 手机预览框+编辑表单并列、实时预览效果 |
| 问答库 | 问答列表 | 左侧项目筛选树、右侧问答卡片列表、展开/收起答案 |
| 效果看板 | 数据面板 | 顶部三个核心指标卡片（启用率/阅读率/风险量）、中部趋势折线图、底部门店排行表 |

### 4.3 响应式设计

- 桌面端优先（1920px/1440px/1280px）
- 侧边导航在 1280px 以下收缩为图标模式
- 卡片网格自适应列数
- 表格在小屏横向滚动
